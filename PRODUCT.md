# Product

## Register

product

## Users
Indonesian marketplace users across four roles: buyers browsing and purchasing products, sellers managing storefronts and fulfilling orders, delivery drivers accepting and completing assignments, and admins monitoring platform health. Context: academic demonstration of a full-stack multi-role e-commerce system with real business logic (tax calculations, SLA enforcement, wallet transactions).

## Product Purpose
SEAPEDIA is a comprehensive multi-merchant marketplace platform connecting buyers, sellers, and delivery drivers. It demonstrates production-grade e-commerce patterns: single-store cart enforcement, PPN 12% tax calculations, voucher/promo systems, driver earnings splits, and SLA-based delivery management with time-leap simulation. Success looks like a polished, fully functional marketplace that showcases professional full-stack engineering.

## Brand Personality
Professional, trustworthy, capable. The tone is confident and competent — a serious marketplace that handles real transactions, not a toy demo. Indonesian-first language with a modern tech sensibility.

## Anti-references
- Generic Bootstrap marketplace templates with default blue buttons
- Overly playful Tokopedia/Shopee clones with cartoon mascots
- Flat, lifeless admin panels with no visual hierarchy
- SaaS-cream landing pages with purple-to-blue gradients

## Design Principles
1. **Role clarity first** — Every screen instantly communicates which role the user is operating in and what actions are available
2. **Transaction confidence** — Financial flows (checkout, wallet, payouts) must feel secure and unambiguous
3. **Operational efficiency** — Sellers and drivers need to complete repetitive tasks fast; minimize clicks for core workflows
4. **Show real data** — Empty states, loading states, and edge cases are designed, not afterthoughts
5. **Indonesian context** — Currency formatting, tax rules, and delivery patterns match local conventions

## Accessibility & Inclusion
WCAG AA minimum. Keyboard navigation for all primary flows. Sufficient color contrast for data-heavy dashboard views. Reduced motion alternatives for animations.
